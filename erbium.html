Internal Computing Platform

Erbium

Erbium is Project Neura's internal computing platform. There are multiple nodes across the environment, but the main node is available to all members, so it is the standard example for connection and file transfer.

Access Pattern

Access is routed through Cloudflare Access. In practice, ordinary SSH-based commands work once you add the required ProxyCommand option.

Example Host

Main node:
main-erbium.projectneura.org

Use the same pattern for other Erbium nodes by replacing the hostname. The main node is shown here because every member can reach it.

-o ProxyCommand="cloudflared access ssh --hostname %h"

How To Use This

Take your normal SSH or SCP command and add the proxy option exactly once. The placeholder %h passes the destination hostname through to Cloudflare Access, so the same pattern works consistently across nodes.

  • Authenticate with Cloudflare Access when prompted.
  • Replace the hostname if you are targeting a different Erbium node.
  • Keep the rest of the command exactly as you would for ordinary SSH tooling.

Why Main Node

The main node is the safest documentation baseline because it is universally accessible to Project Neura members. That makes the examples predictable while still reflecting the same access method used elsewhere in the platform.

SSH Example

Connect to the main node shell through Cloudflare Access.

ssh -o ProxyCommand="cloudflared access ssh --hostname %h" access@main-erbium.projectneura.org

SCP Example

Transfer a local file to the main node using the same access pattern.

scp -o ProxyCommand="cloudflared access ssh --hostname %h" path/to/localfile access@main-erbium.projectneura.org:path/to/remotefile

JupyterLab Access

Users can also access JupyterLab in the browser at https://jupyter-main-erbium.projectneura.org. This provides a web-based interface for notebooks, terminals, and file browsing on the main Erbium node.

When To Use It

Use JupyterLab when you want an interactive development environment without opening a direct SSH session first. It complements the SSH and SCP workflows shown above.

These examples document the access pattern, not an exclusive node list. Replace the hostname with the relevant Erbium node when needed.
